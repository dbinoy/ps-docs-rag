# Central 1 File Specifications - Architecture Brief

Document Scope

This specification set defines the file format standards and operational procedures for clearing files exchanged between Central 1 and member financial institutions via the File and Report Exchange platform. It covers both inbound and outbound file processing.

Key Architectural Elements

File Structure:

  • Files organized in batches (A header record → detail records → Z trailer record)
  • Multiple batches supported per file
  • Business day dating rules apply (excludes weekends, statutory holidays)

Data Flow:

  • Inbound: Member institutions → File and Report Exchange → Central 1 processing
  • Outbound: Central 1 → File and Report Exchange → Member institutions
  • Files retrieved/posted via Client Centre folders

Critical Design Constraints:

  • Strict record formatting (header/detail/trailer structure)
  • Business day calendar compliance for transaction dating
  • File retrieval and posting timing guidelines (Chapter 3)
  • Integration point: File and Report Exchange (platform dependency)

For Architecture Planning

Before designing enhancements, confirm:

1. Current file format version and batch processing rules

2. Posting/retrieval SLAs and timing dependencies

3. Client Centre folder access patterns and authentication

4. Clearing file availability windows and schedules

Reference detailed specs in Chapters 3-5 for specific format requirements.
