# File Exchange and Report Distribution – Architecture Brief

Overview

This document group describes Central 1's File and Report Exchange platform—a dual-channel system for secure file upload/download operations supporting member financial institutions. It encompasses both automated (SFTP/direct connection) and manual (Client Centre portal) exchange mechanisms.

Key Architectural Components

Access Channels:

  • SFTP (direct connection) for automated/scripted integrations
  • Client Centre web portal for interactive file management
  • User authentication/authorization via CPS Admin or User Management systems

Critical Design Elements:

  • Role-based folder structure with access controls tied to user permissions
  • Institutions own responsibility for internal security policies and user management
  • No Central 1 support for client-side FTP scripts—implies stateless, standards-compliant SFTP endpoint

Architect Considerations

1. Security Boundary: Each institution manages their own user access policies; Central 1 enforces authentication but trusts institution-level controls

2. Integration Pattern: Asymmetric channels (automated SFTP vs. portal) require consistent authorization logic across both

3. Operational Constraint: Folder structure appears dynamic/role-dependent—likely requires permission-aware directory listing

4. Support Scope: Central 1 doesn't support custom integrations; design for standard SFTP clients only

Next Steps: Review sections 4 (user access) and 7 (folder structure) for data model specifics before designing API/portal enhancements.
