# Service Charges - Architect Brief

Overview

This document defines the Payment Services Class A Fee Schedule (Form 2725), which establishes product and service pricing for Central 1's Technology and Payment Services Department. It applies to Class A credit unions.

Key Architectural Considerations

Scope & Data Model:

  • Pricing configuration across payment products and services
  • Fee structure likely includes multiple dimensions: product type, transaction volume, service tier, or usage metrics
  • Assumes versioning/effective dating for fee changes across member institutions

Integration Points:

  • Billing/invoicing systems must reference current fee schedules
  • Member-facing portals require accessible pricing lookup
  • Transaction processing may need real-time fee calculation logic

Business Rules to Embed:

  • Member eligibility filtering (Class A credit unions only)
  • Schedule versioning/activation management
  • Potential fee tier logic or volume-based pricing

Design Constraints:

  • Single source of truth for pricing across all downstream systems
  • Change management process for fee updates (effective dating, communication to members)
  • Audit trail requirements for pricing changes

Pre-Design Actions

  • Request complete Form 2725 specification
  • Clarify fee calculation complexity (simple lookup vs. rule-engine required)
  • Identify systems consuming these fees (billing, reporting, member portal)
  • Confirm approval/audit workflow for fee changes
