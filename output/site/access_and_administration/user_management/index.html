# User Management Document Group - Architecture Brief

Overview

This document group covers the Client Centre's user lifecycle management system, including authentication, authorization, user provisioning/deprovisioning, and access control for high-security applications (e.g., Wires).

Key Architectural Components

Authentication & Security:

  • 2-step security tokens required for Security Officers; used for accessing high-risk applications
  • Password registration integrated with Client Centre login
  • Token assignment/management is a critical data flow

Access Control Model:

  • Role-based: Client Centre Users vs. User Management Security Officers (privileged users)
  • Application-level permissions assigned by Security Officers
  • Policy enforcement restricts access based on application sensitivity

User Lifecycle:

  • Creation, modification, deletion of users by Security Officers only
  • Approval workflow required for sensitive changes (name, contact info, permissions)
  • 2-step token lifecycle management (assignment, reset, revocation)

Data Flows:

  • User provisioning → Permission assignment → Application access
  • Audit trail required (CSV reporting capability)
  • Request approval gates for high-impact changes

Critical Design Constraints

  • All Security Officer actions must be auditable
  • 2-step security integration is non-negotiable for privileged access
  • Approval workflows required for user/permission modifications
  • CSV export capability for compliance/reporting

Key Integration Point: User Management is upstream to all Client Centre applications.
