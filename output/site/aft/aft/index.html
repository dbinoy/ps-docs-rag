# AFT Documentation Summary

Overview

This document group covers Originating Automated Funds Transfer (AFT) operations—the core process for initiating electronic fund transfers within the Canadian Payments Association (CPA) framework.

Key Architectural Elements

Core Processes:

  • Originating AFT transaction initiation and submission
  • AFT re-presentment handling (retry/dispute logic)
  • CPA transaction code mappings and validation

Critical Data Flows:

  • Member institution → Central 1 → CPA network
  • Transaction metadata includes CPA codes, amounts, accounts, and settlement instructions
  • Re-presentment triggers and state management for failed transfers

Business Rules:

  • CPA-defined transaction codes govern transfer types and processing rules
  • Service charge application (timing and calculation)
  • Settlement and clearing windows tied to CPA batch schedules

Design Considerations

  • Integration Points: CPA submission format/timing, member bank APIs, clearing/settlement systems
  • Constraints: CPA operational hours, batch processing windows, compliance with payment regulations
  • State Management: Track original vs. re-presented transactions for reconciliation

Before designing enhancements, confirm current transaction volume, SLA requirements, re-presentment failure rates, and whether changes affect CPA submission formats or member bank integrations.
