# AFT Reports Summary for Solutions Architect

  • Purpose: Documents three access mechanisms for AFT (Automated Transfer System) clearing reports generated by Central 1, enabling financial institutions to retrieve transaction data via MERG files, CRF*/ICR1 advance files, or PaymentStream AFT portal.
  • Report Access Channels: MERG file (next business day, consolidated) vs. CRF*/ICR1 files (same-day advance access to CAD clearing reports); CRF*/ICR1 files purged after 7 calendar days; PDFs timestamped to differentiate advance versions from archival MERG copies; reports segregated by currency subfolder (CAD/USD).
  • Key Report Types & Codes: Transaction lifecycle tracked across 20+ report types with dual nomenclature (IC*/UC* for CAD/USD); critical reports include Settlement Register (ICSR/UCSR), Consolidated Returns/Recalls/Rejects (ICRR/UCRR), Item Trace (ICTI/UCTI), and Daily AFT Clearing Settlement (ICPS/UCPS) for reconciliation mapping.
  • Data Structure & Fields: Reports indexed by Originator ID, File Creation Number, Settlement Date, Run number (CRF1-4), and branch; filenames include timestamps (yyyymmdd_hhmm); C/D/E/F/I/J record type designations indicate transaction direction (credit/debit/error correction/return).
  • System Dependencies: Tight integration with File Exchange and Report Distribution infrastructure (FTP-based), CAS Online for settlement posting, PaymentStream AFT for business member self-service; regional variants (AIBC/AIBU for BC, AIOC/AIOU for Ontario via ONDL file).
  • Architectural Constraints: Same-day report availability contingent on CAD AFT clearing run completion; 7-day retention policy for advance reports creates dual archive requirement; account number validation warnings (ICRW/UCRW) sent asynchronously to originators.
  • Integration Point: Business members inputting via PaymentStream AFT retrieve reports directly through portal; financial institutions performing data entry on behalf of clients require custom distribution arrangements configured during onboarding—no standard API documented.
