# AFT Reports - Architecture Brief

Overview

AFT Reports document Automated Funds Transfer (AFT) reporting capabilities provided by Central 1. Reports are distributed via FTP and cover account activity, balances, and transaction details across regional variants (BC: AIBC/AIBU; Ontario: AIOC/AIOU).

Key Data Flows

  • Transaction Types: Debits (PADs - pre-authorized debits) and credits (direct deposits)
  • Report Distribution: FTP-based delivery mechanism
  • Regional Variants: Province-specific report formats and naming conventions

Critical Architecture Considerations

Integration Points:

  • FTP Service for report delivery and retrieval
  • File Exchange and Report Distribution subsystem
  • Account-level transaction data aggregation

Design Constraints:

  • Must support multi-regional compliance (BC/Ontario variants)
  • Preserve existing report nomenclature and format standards
  • Maintain AFT transaction classification integrity (debit vs. credit)

Before Enhancement:

  • Confirm report frequency and latency requirements
  • Clarify which report variants your credit union members require
  • Understand downstream consumers of AFT data (accounting systems, reconciliation, compliance)
  • Validate FTP security and failover mechanisms
  • Document any custom mapping between internal transaction types and AFT classifications

Next Steps: Review full documentation and File Exchange/Report Distribution component specifications.
