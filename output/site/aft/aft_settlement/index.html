# AFT Settlement - Architecture Brief

Overview

This document set defines Automated Funds Transfer (AFT) settlement operations at Central 1, covering both outgoing (PADs/debits and direct deposits/credits) and incoming transaction flows.

Key Architectural Elements

Transaction Types:

  • Outgoing: Pre-authorized debits (PADs) and direct deposits initiated by institutions/businesses
  • Incoming: Transactions received and posted to Central 1 accounts

Settlement Processing:

  • Outgoing files posted per lead time and due date; offsetting transactions applied as applicable
  • Incoming transactions posted by: run, net settlement, branch, and currency
  • Service charges assessed separately on outgoing transactions (CAD/USD)

Critical Data Flows:

  • File submission → validation → posting to Central 1 accounts
  • Multi-dimensional posting logic (branch, currency, settlement method)
  • Charge calculation and application

Design Constraints

  • Regulatory compliance required (legal chapter references statutes/regulations—review before building)
  • Currency-specific handling (CAD/USD minimum)
  • Settlement timing tied to lead times and due dates
  • Net settlement calculations required

Pre-Design Actions: Review section 3 (Legal Considerations) for regulatory obligations affecting validation, posting, and audit requirements.
