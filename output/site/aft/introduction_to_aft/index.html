# AFT Introduction Summary for Solutions Architects

Overview

This document group covers Originating Automated Funds Transfers (AFT) — the operational and compliance framework for electronic credit/debit transactions authorized by account holders. Central 1 acts as the AFT service provider for member financial institutions.

Key Architectural Considerations

Service Lifecycle:

  • Enrollment takes ~2 weeks; involves policy establishment, legal/regulatory compliance, and credential provisioning
  • Service can be dormant or cancelled via formal request processes

Transaction Types & Rules:

  • Primary focus: Pre-authorized Debits (PADs) with four defined categories (Business, Consumer, Healthcare, Utility)
  • Governed by CPA Rule H1 and applicable financial legislation
  • Strict file delivery deadlines tied to Pacific Time zone
  • Record entry limits per submission window

Integration Points:

  • File-based submission model with deadline-driven processing
  • Form-based administrative requests (e.g., dormant period via Form 2044)
  • Payor/Payee/Originator three-party interactions

Security & Compliance:

  • 2-step authentication required for application access
  • Member institutions must enforce KYC, AML, and transaction policies
  • Legal accountability resides with originating institutions

Design Implications

Expect regulatory-driven constraints, deadline-critical processing windows, and need for audit trails. Plan for multi-tenant isolation and strict policy enforcement at origination points.
