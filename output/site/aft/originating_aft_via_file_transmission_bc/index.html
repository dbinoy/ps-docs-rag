# Brief: Originating AFT via File Transmission (BC)

Overview

This document group covers the end-to-end process for credit union originators to submit AFT (Automated Funds Transfer) transactions to Central 1 via file upload when their software lacks native Central 1 support.

Key Architectural Elements

Data Flow:

  • Originators generate AFT files using in-house or third-party software
  • Files are uploaded to Central 1 via defined transmission mechanism
  • Central 1 processes transactions following standardized rules

Process Steps:

1. Setup phase (originator configuration)

2. File creation (originator-side)

3. File upload (transmission to Central 1)

4. Transaction processing (Central 1-side)

Critical Design Constraints:

  • Unified Processing Rules: AFT processing logic is identical regardless of creation method or transmission channel—consolidate business rules in a single service layer
  • File Format Standards: Must support originator software variability; validate format strictly at ingestion
  • Integration Points: File upload mechanism, validation pipeline, and core AFT processor must be decoupled

Architect Considerations

  • Design for asynchronous file processing with clear status tracking
  • Implement robust validation at file intake (format, business rules)
  • Plan for error handling and rejection workflows
  • Consider scalability for concurrent file uploads from multiple originators
