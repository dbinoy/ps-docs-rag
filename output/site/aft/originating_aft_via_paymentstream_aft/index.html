# PaymentStream AFT - Architectural Brief

Overview

PaymentStream AFT is Central 1's Automated Funds Transfer origination platform enabling financial institutions and business members to initiate AFT transactions. The system manages the complete lifecycle from enrollment through settlement.

Critical Architecture Points

Data Flows:

  • Dual ingestion: Direct UI data entry OR file uploads (third-party/in-house software)
  • Payor/payee records → transaction release → Central 1 settlement
  • Downstream reporting from PaymentStream AFT and PaymentStream Direct platforms

Key Integration Constraints:

  • Hard AFT file delivery deadlines (per-transaction timing requirements)
  • Record entry limits (Chapter 13 specifics not fully detailed here)
  • Mandatory approval workflows (conditional: user-level + originator monthly limits)
  • 2-step authentication on all platform access

Governance Requirements:

  • User access via CPS Admin Security Officer or Client Centre User roles
  • Originator information management and policy compliance
  • Risk management: limit enforcement with system alerts
  • Data purge schedule (retention policy critical for compliance)
  • Audit-grade transaction approval trails

Design Constraints:

  • Compliance-first: identification requirements, record-keeping mandates
  • Multi-tenancy: institution + business member originator models
  • Approval state management with role-based authorization
  • Real-time limit monitoring and enforcement

Key Gap: Specific deadline windows, record limits, and purge schedules require review of Chapter 13 detail.
