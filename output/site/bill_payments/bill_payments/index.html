# Bill Payments System Brief

Overview

The Electronic Bill Payments System enables credit union members to pay bills electronically through Central 1's infrastructure. The document outlines operational frameworks, transaction guardrails, and processing windows.

Key Architectural Elements

Transaction Controls:

  • Financial Institution (FI) transaction limits—caps on per-member or per-institution transaction volumes
  • Biller-specific transaction limits—individual payee constraints to manage risk and operational capacity

Processing Windows:

  • Cut-off times define submission deadlines for same-day/next-day processing
  • Critical for scheduling batch jobs, fund settlement, and biller delivery

Design Considerations

Integration Points:

  • Member-facing submission interface
  • Biller directory/validation service
  • Fund settlement and clearing mechanisms
  • Audit/compliance logging for transaction limits

Data Flows:

  • Member payment instruction → validation → queuing → settlement → biller delivery
  • Limit-checking must occur pre-submission to prevent rejections

Constraints:

  • Regulatory compliance requirements around transaction recording
  • Idempotency for retries during settlement failures
  • Real-time limit validation (lookup performance critical)

Before Enhancement Planning:

Confirm current limit enforcement mechanisms, cut-off time handling across time zones, and whether limits are soft (warnings) or hard (blocking).
