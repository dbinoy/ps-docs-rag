# Online Tracing System (OLT) – Architecture Brief

Purpose

The Online Tracing System enables financial institutions to create and manage trace requests for bill payments processed through Central 1's Electronic Bill Payments System. It supports both member-initiated and biller-initiated trace workflows.

Key Data Flows

  • Inbound: Trace requests from Tracers/SuperTracers; biller-initiated requests from Central 1
  • Outbound: Trace status notifications; settlement postings to Current Account Services (CAS)
  • Lifecycle: Created → Processed → Closed → Auto-archived (7 days) → Settled

Critical Rules & Design Constraints

  • User roles: Tracers and SuperTracers (via Client Centre credentials); CPS Admin Security Officers manage security
  • Archival: Automatic after 7 days; manual initiation permitted
  • Settlement: Adjustments and service charges reconciled against CAS postings
  • Reporting: CSV/PDF exports required; used for daily reconciliation
  • Legacy path: Transactions >1 year old bypass OLT (alternate submission mechanism)

Architecture Considerations

  • Multi-environment support (three models offered)
  • Email notification integration
  • Role-based access control tied to Client Centre User Management
  • State-machine workflow (status determines Inbox/Outbox visibility)
  • Reporting and archival as critical operational flows
  • Integration touchpoint: Current Account Services (CAS) for settlement

Key integration dependency: Central 1's Electronic Bill Payments System and CAS.
