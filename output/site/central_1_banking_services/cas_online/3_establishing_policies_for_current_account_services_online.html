# Summary: Posting Advices, Account Statements, and Reports

  • Purpose: Documents three categories of reconciliation reports (adjustment advices, monthly statements, and billing reports) available via Central 1's File and Report Exchange, not generated within CAS Online itself.
  • Report Types & Generation Schedules:
  • CHQ4/ADVC (Credit/Debit Memo Slips): Manual transaction adjustments; CHQ4 for BC transit numbers, ADVC for other regions; used to post general ledger entries
  • STMT (Monthly Statement): Generated night of first business day; contains daily debit/credit transactions with 16-digit max reference field (5-10 chars automated batch, up to 16 manual batch)
  • CHGS (Banking Fees & Charges): Generated by second business day; itemized into three sections (Fees, Item Charges, Deposit Interest) with current/previous month comparisons and volume tracking
  • Key Data Structure: Reference field is critical reconciliation bridge—must match between posting advices and statements; supports both automated batch and manual batch transactions with variable length constraints.
  • Distribution & Storage: Reports stored in financial institution root directories on File and Report Exchange; MERG file format for data, PDFs in Banking folder; regional variations (Ontario vs. BC availability differ).
  • Integration Dependency: Tight coupling with File and Report Exchange distribution system; architects must account for asynchronous report availability and file retrieval workflows separate from CAS Online platform.
  • Reconciliation Pattern: Reports designed to support three-way reconciliation—advices capture manual adjustments, statements provide transaction detail, and billing reports track costs for accurate account balancing.
