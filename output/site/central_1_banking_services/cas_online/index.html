# CAS Online Architecture Brief

Overview

CAS Online is Central 1's near real-time account information portal for member financial institutions. It provides visibility into settlement accounts, transaction history, and posting advices for reconciliation purposes.

Key Data Flows & Architecture Points

Data Refresh Cadence:

  • Account Summary: 20-30 minute refresh interval
  • Transaction History: 1-year rolling window
  • Real-time access model (not batch-driven)

Integration Touchpoints:

  • Direct web link + Client Centre portal integration
  • User management isolated to CAS Online (cannot be provisioned via CPS Admin or User Management systems)
  • Account grouping controls what data is surfaced per user

Critical Business Rules:

  • Branch/Institution Number (5-digit + transit) identifies accounts
  • Business Day definitions apply to transaction dating
  • Access control is institution-managed; Central 1 doesn't enforce per-account restrictions
  • Reports (posting advices, statements) are non-transactional but support reconciliation workflows

Design Constraints

  • Session management requires cookie support
  • User provisioning must be handled independently (no SSO integration documented)
  • Account grouping logic drives UI filtering, not data isolation
  • Must support multi-institution deployments with consistent UX

Consideration: Verify whether "near real-time" has formal SLA requirements before proposing caching or API-based enhancements.
