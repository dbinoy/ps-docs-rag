# CEBA Loan Repayments Architecture Brief

Document Scope

This document set covers the end-to-end lifecycle for Canada Emergency Business Account (CEBA) loan administration within Central 1's payment infrastructure, specifically addressing repayment processing, fee handling, settlement operations, and reconciliation workflows.

Key Architectural Components

Core Processes:

  • Repayment capture and validation
  • Administration fee calculation and application
  • Settlement batch processing
  • Reconciliation workflows (loan-to-payment matching)

Data Flow Considerations:

  • Repayments flow from member credit unions → Central 1 → originating loan accounts
  • Administration fees generated per transaction or batch cycle
  • Settlement timing constraints (likely T+1 or batch-dependent)
  • Reconciliation requires bidirectional audit trails for dispute resolution

Critical Design Constraints

  • Regulatory compliance: CEBA is government-backed; audit trails and reporting are mandatory
  • Multi-party settlement: Funds flow through Central 1 as intermediary; settlement timing affects liquidity
  • Fee application logic: Must handle variable fee structures across member institutions
  • Idempotency: Repayment reversals/corrections require robust transaction handling

Pre-Design Questions

  • Current settlement frequency and SLA?
  • Existing reconciliation automation level?
  • Fee calculation variability by member?
