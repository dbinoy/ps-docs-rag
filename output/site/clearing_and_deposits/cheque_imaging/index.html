# Cheque Imaging System – Architecture Brief

Overview

Central 1's Cheque Imaging system captures and manages digital images of deposit items, chargebacks, and incoming clearing items. Physical cheques are retained only for Unqualified Items and Chargebacks; all others are imaged.

Key Data Flows & Integration Points

  • Retrieval: Online Return System (ORS) enables search, view, and print of Incoming Clearing Items and deposit items
  • Member Delivery: Image files provided to members via statements (excludes Unqualified Items and Chargebacks)
  • MICR Testing: Validation service for cheque MICR encoding on new/existing accounts

Critical Design Constraints

  • Legal Compliance: System must adhere to applicable financial services legislation and regulations (Chapter 3 details requirements)
  • Transit Number Components: Branch Number (5-digit) + Institution Number = Transit Number (required for clearing operations)
  • Physical Retention Rules: Physical cheques retained only for Unqualified Items and Chargebacks; all others digitized
  • ORS as Primary Tool: Prefer ORS resolution for inquiries before alternative escalation paths

Architecture Considerations

  • Separate data retention policies by item type
  • MICR validation as upstream quality gate
  • Image lifecycle management tied to item classification
  • Legal audit trail requirements for regulatory compliance
