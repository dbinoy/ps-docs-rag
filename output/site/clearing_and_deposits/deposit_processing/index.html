# Deposit Processing - Architecture Brief

Overview

Central 1's Deposit Processing documentation covers end-to-end handling of paper and remote deposit items (cheques, drafts, money orders, warrants) through three channels: Branch Capture, Corporate Capture, and Mobile Deposits.

Key Data Flows

1. Capture → Review → Approval → Transmission (Branch/Corporate)

2. Mobile deposit → Customer submission → Central 1 processing

3. Settlement → T+0 or T+1 credit to institution account based on cut-off times

4. Aged deposits → Auto-escalation for deposits >24 hours untransmitted

Critical Architecture Elements

  • Image capture & storage: MICR-encoded items (Canadian/US dollars); deposit slip images retained
  • Multi-user workflow: Role-based access (Capture Users, Reviewers, Supervisors, Admins) via Client Centre
  • Integration points: Paystation (scanner procurement portal), Online Return portal (image retrieval)
  • Regulatory compliance: Legal holds, identification verification, record retention
  • Research capability: Deposit tracing, reconciliation, reporting against Central 1 account

Design Constraints

  • 24-hour transmission SLA (triggers aged deposit workflows)
  • Staging environment required for testing
  • Settlement reconciliation dependency on Central 1's backend
  • Collection-only items (foreign non-USD) require manual handling
