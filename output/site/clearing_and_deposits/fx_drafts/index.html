# FX Drafts System Architecture Brief

Overview

FX Drafts is a Central 1–managed, in-branch draft issuance platform supporting 20+ foreign currencies (including USD). Users create and print negotiable drafts for beneficiaries through a web-based application integrated with Client Centre.

Key Architectural Components

Participants:

  • Central 1 (operator/settlement hub)
  • Convera (technology/processing partner)
  • Member financial institutions (issuers)
  • Client Centre (authentication/authorization layer)

Data Flows:

  • User access provisioned via Client Centre user management (role-based: Security Officer minimum)
  • Draft creation → in-branch printing
  • End-of-day settlement: Convera generates settlement file → Central 1 debits institution's CAD account
  • Stop payments and stock orders route to Convera directly (with contingency contact procedure)

Critical Design Constraints

1. Compliance-heavy: PCMLTFA/PCMLTF regulations mandate staff training policies; AML officer involvement required

2. Multi-party agreements: Institutions must sign agreements with both Convera and Central 1 before enrollment

3. Settlement dependency: Daily file-based reconciliation; no real-time settlement

4. Contingency requirement: System downtime protocol mandates direct Convera contact

5. Authentication: Dual-layer (Client Centre + FX Drafts) access control

Before designing enhancements, clarify: settlement SLA, file format specifications, failover escalation workflows, and AML integration touch-points.
