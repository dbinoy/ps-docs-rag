# ICBC Agents Deposits (BC) - Architecture Brief

Overview

This document group defines the operational and settlement framework for deposit accounts held by ICBC insurance agents at member credit unions. Central 1 acts as the clearing and settlement intermediary.

Key Data Flows

1. Deposit Preparation: Agents separate cheques/cash, stamp cheques (ICBC Agent Stamp, upper right quadrant back)

2. Deposit Processing: Member institutions receive deposits, credit ICBC agent accounts, report details to Central 1

3. Settlement: Central 1 credits member institutions' Central 1 accounts for cheque values; processes via branch capture

4. Sweep Mechanism: Central 1 initiates PADs (pre-authorized debits) against agent accounts to transfer funds

Critical Rules & Constraints

  • Account Type: Deposit-taking only (no overdraft/credit functions)
  • Identification: Transit Number (Institution + 5-digit Branch Number) required for routing
  • Chargeback Handling: Misdirected chargebacks must be managed; dishonoured PADs require exception handling
  • Reconciliation: Account errors and discrepancies must be tracked and resolved

Architecture Considerations

  • Design for dual-ledger reconciliation (member institution ↔ Central 1)
  • PAD failure handling and retry logic critical
  • Cheque imaging/capture integration required
  • Clear separation between agent deposits and standard deposits in GL
