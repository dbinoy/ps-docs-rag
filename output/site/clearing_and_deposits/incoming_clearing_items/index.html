# Incoming Clearing Items – Architecture Brief

Overview

This document group defines Central 1's processing framework for Canadian and US dollar incoming clearing items (cheques and similar instruments) received by credit unions. It covers the complete lifecycle: receipt, processing, settlement, and reporting.

Key Data Flows & Integration Points

Inbound: Items drawn on credit unions or members enter the clearing cycle post-deposit/cash.

Processing: Items move through clearing with validation against branch/institution identifiers (5-digit branch number + institution number = transit routing).

Settlement: Item totals (including chargebacks) are debited to the credit union's Central 1 account; settlement reconciliation is core to the process.

Reporting: Multi-report outputs available via MERG file downloads and ORS (Online Reporting System) for search, viewing, and tracing.

Critical Architecture Considerations

  • Regulatory compliance is mandatory—document references applicable statutes (Chapter 3)
  • Dual currency support (CAD/USD) with distinct processing paths
  • Chargeback handling integrated into settlement logic
  • Audit trail requirements via ORS querying and print capabilities
  • Reconciliation criticality—settlement accuracy directly impacts member accounts

Design Constraints

  • Use ORS as primary inquiry/resolution tool (avoid parallel systems)
  • Maintain audit-grade traceability for compliance
  • Support real-time and batch reporting modes
