# Provincial Concentrator Account Deposits (BC) – Architecture Brief

Overview

This document group defines the operational and technical framework for managing BC provincial government deposit concentrator accounts through Central 1 Credit Union's clearing and settlement infrastructure.

Key Architectural Elements

Account Structure:

  • Concentrator holding accounts opened by Province of BC or authorized agents
  • Operating instructions delivered via new account letters
  • Requires Transit Number (Institution + 5-digit Branch Number) for routing

Core Processing Flow:

1. Deposits received (cheques, PADs, business remittances)

2. Central 1 processes and validates deposits

3. Automated sweep via Pre-Authorized Debits (PADs) to concentrate funds

4. Settlement via Central 1 member account credits

Critical Business Rules:

  • PAD-based sweep mechanism for fund concentration
  • Dishonoured item handling with chargeback processing
  • Account error reconciliation procedures
  • Branch capture cheque settlement with explicit credit timing

Integration Points:

  • PAD creation/management (automated sweeps)
  • Chargeback dispute handling
  • Transit number validation
  • Settlement notification to member institutions

Design Constraints:

  • Compliance with provincial government operating instructions
  • Error handling for dishonoured transactions
  • Multi-entity routing (BC Services offices, authorized businesses)
  • Clear settlement audit trail

For Enhancement Planning: Map PAD orchestration, error queue handling, and settlement confirmation workflows.
