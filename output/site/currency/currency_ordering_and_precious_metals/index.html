# Currency Ordering and Precious Metals – Architecture Brief

Overview

This document set describes Central 1's end-to-end cash management system, covering currency ordering (CAD/USD/FX), returns, precious metals, and associated settlement processes for member credit unions.

Key Architectural Components

Access & Authentication:

  • Client Centre login via Central 1 (password-based)
  • Role-based access: CPS Admin Security Officers, Admin Users, Regular Users
  • Duty segregation enforced at policy level

Core Systems:

  • Cash Services application – order creation, management, reporting
  • ServiceNow – standing orders, late orders, contingency orders (with fallback procedures when unavailable)
  • Globex 2000 integration – third-party FX Notes provider

Key Data Elements:

  • Branch number (5-digit identifier + institution number = transit number)
  • Daily ordering limits per location (managed in ServiceNow)
  • Denomination/quantity constraints for CAD, USD, FX
  • Settlement date = dispatch date (with branch-level charge debiting)

Critical Design Constraints

  • Maximum order threshold defined per transaction type
  • Fallback mechanisms required when ServiceNow is unavailable
  • Multi-location support with location-specific limits and access controls
  • Third-party dependencies (Westkey Graphics for return supplies; Globex 2000 for FX)
  • Audit trail requirements via Order History and reports

Integration Points: Client Centre authentication, ServiceNow workflows, Globex API, settlement systems.
