# FX Notes Architecture Brief

Overview

FX Notes is a Central 1 platform enabling authorized credit union users to buy/sell foreign currency bank notes (not coins) via partnership with Globex 2000. The system is web-based, accessed through Client Centre authentication.

Key Architecture Components

Access & Authentication:

  • Role-based access (Standard Users, Managers, Auditors, CPS Admin Security Officers)
  • Integrated with Client Centre User ID system
  • Segregated duty model enforced at policy level

Core Transactions:

  • Sell flow: User enters order → Globex 2000 receives → fulfillment
  • Buy flow: Authenticity verification required → order entry → settlement
  • Net settlement in CAD to institution accounts

Data & Operations:

  • Business day definition: Mon-Fri excluding statutory/Quebec holidays
  • 5-digit branch identifiers for multi-branch operations
  • 25-month report retention
  • Configurable FX spreads (retail rate modifiers) by institution

Settlement:

  • Centralized net settlement model (BC/Ontario)
  • Integration point with institution's general ledger

Design Considerations

  • Audit trail requirements (transaction images/shipments viewable)
  • Regulatory compliance architecture (financial institution responsibility)
  • Multi-currency support with dynamic rate management
  • Branch-level transaction segregation and reporting
