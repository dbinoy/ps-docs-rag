# Data Hub Section Overview

Purpose

The Data Hub section documents Central 1's analytics platform—a Microsoft Power BI-based dashboard system that enables financial institutions to monitor payment performance, consumption patterns, and KPIs. This section covers platform access, user provisioning, navigation, and data export capabilities. It establishes the foundational architecture for delivering analytics-as-a-service to credit union members.

Key Concepts

1. Data Hub Platform: A multi-tenant, cloud-hosted analytics environment accessed through the Client Centre portal. Provides interactive visualization, filtering, and benchmarking against anonymized peer cohorts.

2. Payments Dashboard: The primary analytics interface within Data Hub. Supports multi-level filtering, chart/table views, performance comparison to peers of similar size, and structured data export.

3. Client Centre Authentication: Centralized identity management portal (https://clients.central1.com) serving as the single entry point. First-time users receive username prompts before credential entry.

4. User Provisioning Model: Central 1 owns all user lifecycle management. Member institutions cannot self-serve via CPS Admin or User Management applications; all requests route through ServiceNow.

5. Data Export: Two pathways—static image export to PowerPoint and live data export using dropdown menus. Export scope and format determined by user role and dashboard design.

6. Vendor Stack: Underlying platform is Microsoft Power BI. Standard Power BI features are accessible but unsupported by Central 1 engineering.

How It Works

1. User authenticates via Client Centre portal with Central 1-managed credentials.

2. User navigates to Data Hub application within authenticated session.

3. User applies multi-level filters to Payments Dashboard (institution-scoped data).

4. Dashboard renders visualizations with peer benchmarking context.

5. User exports results via PowerPoint (image) or live data dropdown (raw export).

6. Central 1 retains analytics data; member institution receives derived insights only.

Integration Points

  • Client Centre Portal: Identity provider and application launcher for all Data Hub access.
  • ServiceNow: User provisioning request intake system; Central 1 backend provisioning system.
  • Microsoft Power BI: Underlying visualization and analytics engine; not directly managed by member institutions.
  • Payment Transaction Data: Implicit upstream connection (not detailed in this section); Payments Dashboard consumes aggregated, institution-scoped transaction data.

Architect Notes

  • User Management Constraint: No federated identity or self-service provisioning. All requests create operational dependency on Central 1 ServiceNow queue—plan for provisioning lead times.
  • Vendor Lock-in: Power BI is embedded. Custom dashboards or API-driven integrations are not mentioned; confirm Power BI extensibility SLAs before designing downstream analytics pipelines.
  • Data Residency & Export Controls: Section does not specify data residency, encryption in transit/rest, or export audit trails. Clarify compliance posture before handling regulated datasets.
  • Unsupported Features: Power BI's native features are explicitly unsupported. Scope Data Hub as read-only analytics only; avoid custom calculations or advanced Power BI features in production designs.
