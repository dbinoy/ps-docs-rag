# Enterprise Fraud Management (EFM) - Architecture Brief

Overview

EFM is Central 1's real-time fraud detection and incident management system for member financial institutions. It generates alerts on suspicious transactions, enabling fraud teams to investigate and respond.

Key Data Flows & Integration Points

  • Transaction input: Monitored transactions feed into detection rules
  • Alert generation: Suspicious activity triggers alerts grouped within incidents
  • Notification: Email alerts notify configured fraud teams
  • Account mapping: Account Entity ID links customer accounts to FI identifiers (FI ID, Transit #, Account #)

Critical Business Rules

  • Limit Groups: Risk-based transaction limits for Interac® e-Transfer (requires FI risk assessment)
  • Incident filtering: Configurable filters apply across incident lists for triage
  • Settlement integration: e-Transfer and wire settlement flows managed separately (external settlement documents referenced)

Architecture Constraints

  • Access control: Role-based via CPS Admin/User Management; Client Centre authentication required
  • Multi-tenant: Supports multiple FIs with isolated data views
  • Reporting: Multiple report types (transaction alerts, rule triggers, model summaries) for audit/compliance
  • User management: Centralized administration of fraud team access

Design Considerations

Ensure audit trails on incident status changes, support configurable alert routing, maintain data isolation by FI/Account Entity ID, and plan for high-volume transaction processing at detection layer.
