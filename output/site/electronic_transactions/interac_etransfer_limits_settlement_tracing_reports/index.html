# Interac e-Transfer: Architecture Brief

Purpose

This document set governs transaction limits, settlement mechanics, reconciliation, remediation, and reporting for Interac e-Transfer® operations at Central 1 Credit Union.

Key Architectural Elements

Limit Framework:

  • Per-transaction, 24-hour, and 7-day maximums applied via limit groups
  • Autodeposit capability bypasses manual acceptance but subject to compliance controls

Settlement & Reconciliation:

  • All completed transactions post to Central 1's account with separate totals (sends vs. receives)
  • Daily reconciliation required; failures trigger remediation workflows
  • Timeout errors between Central 1 and banking hosts require asynchronous recovery mechanisms

Critical Data Flows:

1. Transaction → Central 1 → Banking Host (with response code mapping)

2. Settlement batches → Daily reporting (EMCU, EMCD, ECCX)

3. Failed transactions → Remediation queue → Host retry/reversal

Compliance Integration:

  • PCMLTFA/PCMLTF regulations embedded in transaction validation
  • Tracing workflow requires pre-consultation of settlement reports

Design Constraints:

  • Response code translation layer required between Interac and banking systems
  • Idempotency critical for timeout recovery
  • Daily report generation and monthly billing reconciliation are dependencies

Key Consideration: Remediation process is asynchronous; design for eventual consistency with audit trail preservation.
