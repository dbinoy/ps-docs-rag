# Interac eTransfer OAS - Architecture Brief

Purpose

Central 1's Interac eTransfer Online Administration System (OAS) is a Client Centre-hosted application for managing eTransfer transactions, customer profiles, fraud incidents, and transaction tracing across a 2-year lookback window.

Key Integration Points & Data Flows

  • ANR (Account Number Routing): Enables direct ACH-style payments using Institution Number + Transit + Account details vs. email-based transfers
  • Customer Profile Management: Track enabled/disabled states, scrubbed email addresses, and transaction eligibility
  • Tracing & Recovery: Query historical transactions; recover misdirected funds (critical for ANR implementations)
  • Fraud Workflows: Dedicated analyst role for incident management with search/view capabilities

Critical Business Rules

  • Two-level access control required (Client Centre User ID + OAS-specific permissions)
  • Scrubbed email revocation process prevents legitimate addresses from being permanently blocked
  • Profile status (enabled/disabled) gates send/receive capability
  • ANR creates misdirection risk requiring recovery procedures

Architectural Constraints

  • 2-year transaction data retention window
  • Client Centre dependency (not standalone system)
  • Role-based access (CSR, Fraud Analyst)
  • Interac-maintained scrubbed email list integration

Design Consideration: Any enhancement touching ANR, profile state changes, or fraud workflows requires careful audit logging and approval workflows.
