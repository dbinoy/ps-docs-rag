# PaymentStream Direct – Architecture Brief

Overview

PaymentStream Direct is Central 1's unified payment platform UI providing access to core payment products, customer profile management, and MemberDirect system support. It serves as the primary transaction interface for enrolled financial institutions.

Key Architectural Components

  • User Management: Role-based access via CPS Admin/User Management; separate application permissions per function
  • Authentication: 2-step security (OTP-based step-up authentication) required for login
  • Core Products: Bill Payments, AFT (ACH/wire equivalents), Wires (domestic/international), CRA Business Taxes, Lock-n-Block debit card management
  • Third-Party Integrations: ClickSWITCH (Switch Concierge for payment/deposit migration)

Critical Design Constraints

1. Banking System Dependency: Customer profile and bill payment functions require active connection to institution's core banking system

2. Policy/Controls: Each institution responsible for implementing internal controls; no shared security model documented

3. Data Scope: Handles financial transactions, archived task audit trails, and CRA filing submissions

4. User Classes: Client Centre Users and CPS Admin Security Officers only

Pre-Design Considerations

  • Enrollment process (contact Central 1 Client Support) impacts go-live timelines
  • Multi-product architecture suggests loosely coupled microservices; confirm integration patterns
  • Clarify data retention for archived tasks and transaction audit requirements
  • Verify regulatory compliance scope for cross-border wires and CRA submissions
