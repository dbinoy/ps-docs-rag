# Wires Application - Architectural Summary

Overview

The Wires application on PaymentStream Direct™ enables domestic and international wire transfers for authorized credit union users. It manages the full lifecycle: creation, approval, settlement, and reconciliation of outgoing and incoming wires.

Key Data Flows & Integration Points

  • Outgoing wires: User creation → MTS-based approval workflow (1-2 step) → settlement to Central 1 current account (near real-time)
  • Incoming wires: External FI uploads processed every ~10 minutes; real-time for participating institutions
  • FX operations: Live rate feeds; spreads and fees configurable per user permission
  • Settlement: Processed within minutes of approval to Central 1 current account

Critical Business Rules

  • Approval tiers defined by branch settings (amount-based)
  • Editable only pre-approval; post-approval requires recall/trace/amend procedures
  • IBAN/country-specific validation for international wires
  • 2-step security authentication required
  • Templates support recurring wire patterns

Architectural Constraints

  • MTS (Money Transfer System) is the permission/admin control plane—essential dependency
  • Three processing models available; selection affects workflow design
  • Contingency procedures require designated backup processors
  • Inter-branch transfers require currency alignment
  • Legal/regulatory compliance mandatory per jurisdiction

Key Integration Points

Client Centre authentication → MTS admin/permissions → Wires application → Settlement backend
