# Payment Solutions 20:41 Release Summary

- **Release scope**: October 27, 2020 update to CEBA (Canada Emergency Business Account) application procedures, specifically enabling CEBA 4.0 eligibility for previously ineligible business customers who operate via personal accounts.

- **Eligibility expansion**: CEBA 4.0 targets business customers previously disqualified due to using personal banking accounts; now eligible upon opening a business account—represents a significant policy change affecting customer segmentation and application workflows.

- **ServiceNow integration point**: Updates apply specifically to financial institutions using ServiceNow Workflow; architects must account for workflow configuration changes in that system.

- **New data field requirement**: A new field added to the CEBA application form to flag CEBA 4.0 applicants at the financial institution level—requires schema updates and form handling logic to capture and route this classification.

- **Documentation reference**: Changes documented in "Central 1 Facilitated CEBA Application Procedures – Quick Reference Guide" (Status: 9140, Section 2.2)—architects should reference this for detailed procedure specifications and validation rules.

- **No breaking changes noted**: The update is marked as "Revised" (not deprecated/removed), suggesting backward compatibility with existing CEBA application flows.

- **Support/escalation path**: Client Support Services (1-888-889-7878 Option 2) and digitalbanking_support@central1.com are support contacts for implementation questions.
