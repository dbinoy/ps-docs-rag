# Release 21:32 (September 15, 2021) Summary

- **Purpose**: Documentation of the decommissioning of Customer Automated Funds Transfer (CAFT) system and migration of all Originators to PaymentStream as the replacement platform.

- **System Migration**: Complete sunsetting of CAFT with removal of all CAFT-related procedures and documentation from the Payment Solutions Manual; PaymentStream is the successor system for Originator fund transfer operations.

- **Regional Service Clarification**: AFT (Automated Funds Transfer) via FTP service scope was explicitly defined to cover BC and Atlantic region (not BC-only as previously documented); affected forms were revised for accuracy.

- **Obsolete Documentation Removed**:
  - CAFT Procedures (Ontario) section
  - Originating AFT via CAFT (BC) section
  - Four CAFT-related forms marked obsolete: User Request (1797), Originator ID and User Access Request (2473), System Access Agreement (2642), User Guide (2762)

- **Active AFT Service Forms**: Two FTP-based AFT service documents remain current and were revised—Application for AFT Service via FTP (1378) and File Limit Change Request AFT Service via FTP (1641)—indicating FTP remains a supported integration method for specific regions.

- **Dependency Note**: Any system integration or enhancement work must account for PaymentStream as the active platform; CAFT-dependent integrations or workflows are no longer supported or documented.
