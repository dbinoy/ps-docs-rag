# Payment Solutions Release 23:08 Summary

- **Purpose**: Documentation corrections addressing legacy banking system discrepancies in transaction type codes and AFT return file specifications (effective May 18, 2023)

- **Critical Code Mapping Change**: USD Month End transaction type corrected from code "10" to code "12" in PREC file specifications—legacy system incorrectly displayed "10", documentation now aligns with actual specification

- **CAD vs. USD Distinction**: Data Element 7 in P records now explicitly clarifies that code "10" = CAD month-end charges and code "12" = USD month-end charges to prevent currency-based processing errors

- **AFT Return Clearing File Structure Updates**: 
  - Data Element 10 (I and J records) respecified as "Transaction type from original transaction" rather than "Zero Filled"—affects data mapping logic in return file processing
  - Data Element 20 now requires left-justification formatting in both CAD (Form 9085) and USD (Form 9086) variants

- **No Operational Impact**: Corrections are documentation-only; actual AFT Return Clearing Files generated by Central 1 remain unchanged, indicating these were clarifications of existing implementation rather than behavioral fixes

- **Multi-Region File Variants**: Separate specifications maintained for CAD (9085) and USD (9086) with region exclusions (BC and Atlantic), requiring architecture to support parallel processing paths

- **File Format Precision**: USD AFT specifications now explicitly define ".DAT" file extension in documentation (Section 3, Form 9086), a constraint for downstream integration points
