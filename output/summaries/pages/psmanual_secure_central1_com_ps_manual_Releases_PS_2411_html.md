# Release 24:11 Summary for Solutions Architects

- **Release scope**: February 29, 2024 update consolidating scanner installation, maintenance, and troubleshooting documentation for Branch Capture and Corporate Capture into the Deposit Processing Part, with corresponding document obsolescence.

- **Documentation reorganization**: Scanner-related procedures previously distributed across standalone documents (5937, 5928) have been incorporated into Chapters 7-8 of the Deposit Processing Part; this represents a documentation architecture change, not a functional system change.

- **New reporting capabilities**: Two billing-related scanner count reports introduced (BL01 for Branch, RL01 for Corporate in Chapters 14.7-14.8), suggesting scanner inventory tracking is now a billable/auditable data point in the deposit processing workflow.

- **Mobile Deposit integration point**: Mobile Deposit File Specifications document revised with form description updates (doc 3604), indicating mobile deposit submissions interface with the deposit processing system and may require specification alignment during integrations.

- **Branch Capture process dependencies**: Central 1's Concentrator Account deposit processing procedures were updated (doc 5976), indicating tight coupling between Branch Capture scanner operations and Central 1's downstream concentration services.

- **Scanner driver versioning**: Both Corporate Capture (doc 5933) and Branch Capture driver installation documents were revised to include "Updating" procedures, suggesting scanner driver versions are independently managed and may require coordinated rollout strategies across branch and corporate deployments.

- **Support and escalation**: Client Support Services (1-888-889-7878, Option 1) owns first-line support for deposit processing questions; architects should reference this for integration support contacts.
