# PS 24:45 Release Summary – Wires Data Extract Service Request (Form 2592)

- **Overview**: Form 2592 (Wires Data Extract Service Request) was updated December 17, 2024 with content revisions, UI improvements, and e-signature capability.

- **Data Scope Constraint**: The extract includes **all incoming and outgoing wires** with no filtering option; financial institutions cannot selectively include only cross-border wires. This is a fixed data scope requirement.

- **Removed Feature**: Setup Information section was removed from the form as it no longer applies—this represents a simplification of the service request workflow.

- **UI/UX Changes**: Font size increased and readability improved; this is a presentation-layer update with no backend data model impact.

- **E-Signature Integration**: Form now supports e-signatures via the **Central 1 e-Signature Process and Approval Statement**; architects must account for external e-signature system integration and workflow dependencies.

- **Compliance Note**: Financial institutions must validate their own e-signing policies before implementation—this is a client-side governance constraint, not a technical requirement, but affects deployment eligibility.

- **Support Contact**: Form status and technical questions route to Client Support Services (1-888-889-7878, Option 1 or support@central1.com).
