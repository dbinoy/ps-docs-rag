# Payment Solutions Documentation Update Summary – September 2020

- **Purpose**: Release notes documenting four documentation revisions to the Payment Solutions Manual across admin, international transfers, and FX operations domains.

- **Administrative Process Updates**: CPS Admin Modification/Deletion Request (doc 2035) now includes "Add a 2-step security token" as a modification type and renames "Update User Name" to "Change the User's name"—reflects enhanced security token provisioning workflow in the admin modification request data structure.

- **Contact Information Consolidation**: Saskatchewan Credit Unions contact details were updated across multiple documents (CPS Admin Appointment Certificate 1918, CPS Admin Modification/Deletion Request 2035)—indicates potential contact registry dependency or data governance consolidation across regional credit union networks.

- **International Transfers Scope**: Two documents revised (Implementation Requirements 9080 and FX Spread 9111)—suggests international transfers is a distinct subsystem with separate implementation and pricing logic that shares contact information management with admin functions.

- **Critical Integration Point**: All documentation updates reference centralized contact information management (Saskatchewan Credit Unions specifically)—architect must understand whether contact data is maintained in a separate system or embedded within document definitions.

- **Support Escalation Path**: Client Support Services (1-888-889-7878 Option 1, support@central1.com) is the single point of contact for manual clarifications—no indication of automated documentation versioning or change notification system.

- **Architectural Gap**: No versioning scheme, API references, or system integration diagrams are mentioned—suggests documentation portal may be separate from automated deployment/configuration pipelines requiring manual synchronization.
